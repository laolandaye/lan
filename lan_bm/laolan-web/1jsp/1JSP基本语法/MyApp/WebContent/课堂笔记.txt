JavaWeb
一、JavaWeb（JSP/Servlet）基础。
JSP是什么？？
Java Server Page
通俗的来说，JSP技术就是用Java语言来开发网站的。
浏览器/服务器（B/S）模式
客户端/服务器（C/S）模式

C/S模式通常需要制作一个Client。
程序虽然数据是存放在服务器上，但是必须有一个客户端软件。
客户端软件升级就不那么方便了。需要所有的客户端的计算机完全更新客户端的软件。
有部分的功能是在客户端上的，自然就给了程序留了一个漏洞。
富客户端。
因为有部分的功能是在客户端上完成的。所以可以随着客户端安装大量的相关资源。
（贴图，模型，声音，视频……）
占用的客户端的系统资源。

B/S模式就不需要去安装一个客户端，但是需要一个浏览器。
虽然也安装了一个软件，但是客户端的浏览器是一个通用的软件。
升级无需改变客户端浏览器的任何资料。所有的升级工作，全部在服务端完成。
所有的功能完全在服务器上。相对而言，服务器的安全级别要高于客户端计算机。
瘦客户端。
功能都不在客户端上。做华丽的游戏，不是B/S结构应用程序的长项。

端游（C/S结构的）/页游（B/S结构的）

客户端应用程序（简称客户端）：向服务器提交请求，并且接收服务器响应回来的结果的一个程序。
服务端应用程序（简称服务端）：完成向客户端提供数据、功能的请求的处理，并将处理的结果响应回对应的客户端的一个程序。
浏览器：网页的查看工具。在Web程序中，浏览器主要通过HTML实现一个用户界面的功能。

统称为软件
操作系统：Windows、Linux、MacOS……
  提供管理和维护其他软件，以及其他软件与硬件之间的交互。
应用程序：Office、WPS、QQ、端游客户端……
  提供有针对性功能的软件。
桌面应用程序：端游的客户端、开发工具eclipse、IDEA，办公软件、通讯软件……
  采用C/S结构开发，或者是无需网络功能的，运行在桌面系统上的应用程序。
Web应用程序：页游、电子商务、电子政务、娱乐网站、门户网站、搜索引擎、各种管理系统……
  采用B/S结构开发，基于HTML网页形式的应用程序。（开发过程中需要重点关注如何生成这些网页）

服务器软件：服务器的操作系统，提供管理和维护Web应用程序的。通常也可以叫做服务器环境。
MySQL：数据库服务器的软件。
Tomcat：提供Web应用程序的管理和维护工作。也可以叫做Web容器。IIS（apsx）、Apache（html、php）、JBoss（ejb）……
服务器：安装有服务器软件的计算机。

tomcat的安装和配置
1、安装就是解压缩。
2、运行
  2.1、Tomcat的目录结构：
    bin存放的是tomcat的相关的程序文件，批处理
    conf存放的是tomcat相关的配置文件
    lib存放的是tomcat需要使用到的第三方开发包（类库*.jar）
    logs存放的是tomcat运行过程中的日志文件（*.log）
    temp存放的是tomcat运行过程中的临时文件（*.tmp）
    webapps存放的是Web应用程序（理解成为Windows操作系统的Program Files、Program Files(x86)）
    work存放的是Web应用程序运行过程中产生的临时文件
  2.2、Web应用程序在Tomcat中如何运行起来？
    Web应用程序在Tomcat的webapps文件中，每一个Web应用程序是一个文件夹。
    每一个Web应用程序都有一个关键的子文件夹WEB-INF，存放的是Web应用的相关配置文件以及Java类，类库
    WEB-INF这个文件夹名称全部是大写字母。并且是一个隐藏文件夹，通常无法通过浏览器直接访问！！！
    在WEB-INF文件夹中还有一个重要的文件web.xml，这一个文件是Web应用程序的配置文件！！！
    web.xml文件名全小写！！！
  2.3、Tomcat的端口号：8080


网页的两种打开方式：
本地访问:file://
远程访问:http://

JSP文件不支持本地访问方式！！！

Web应用程序通过URL（统一资源定位器）来访问
http://www.baidu.com:80/index.xxx
http://localhost:8080/MyApp/helloJSP.jsp

URL的组成：
协议://计算机名:端口号/Web应用程序名/资源路径/资源名称.xxx
MySQL连接字符串：jdbc:mysql://localhost:3306/test
计算机名包括机器名、IP地址、域名

localhost表示本地计算机的域名

127.0.0.1表示本地计算机的IP
Web应用程序的默认端口号:80，默认端口可以省略

tomcat的启动命令：startup.bat
tomcat的关闭命令：shutdown.bat

查询本机IP：ipconfig /all
查询本机已经打开的端口：netstat -an
或者：netstat -abn
